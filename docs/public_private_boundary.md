# Public/private boundary

## Purpose

This document defines what can be included in this public repository and what must remain outside it.

The repository is intended to provide a public protocol, synthetic-data examples and workflow scaffolds for a post-PhD research programme. It is not intended to release protected data, reproduce controlled-access analyses, or expose unpublished individual-level outputs.

## Public materials

The following materials are appropriate for this repository:

- protocol documents;
- synthetic-data examples;
- general workflow templates;
- validation hierarchy notes;
- dataset access plans;
- public annotation resource notes;
- technical notes;
- schematic figures;
- code that can run without controlled-access data.

## Private or excluded materials

The following materials should not be uploaded to this repository:

- UK Biobank individual-level data;
- ADNI, PPMI, AMP-PD, NIAGADS or other controlled-access individual-level data;
- participant identifiers;
- genotype files;
- imaging files;
- private component score files;
- protected phenotype tables;
- unpublished individual-level results;
- raw outputs from controlled-access analyses;
- supervisor or collaborator correspondence;
- access keys, credentials or tokens;
- local configuration files containing private paths.

## Synthetic examples

Synthetic examples may be used to demonstrate the structure of an analysis.

Synthetic data should be clearly labelled as synthetic and should not be designed to reproduce, approximate or reverse-engineer protected individual-level data.

## Controlled-access analyses

Analyses using controlled-access datasets should be conducted in separate private workspaces that follow the relevant data-use agreements.

Public outputs from such analyses should only be added here if they are permitted by the relevant agreements and have been checked for disclosure risk.

## Practical rule

If a file depends on controlled-access individual-level data, it should stay out of this public repository unless there is a clear reason, documented permission and no disclosure risk.
